# bot_1st

[🔗English dock](README.md)

یک ربات پایتون برای مدیریت سفارش‌ها و پروژه‌های توسعه ربات تلگرام / بله با پنل کاربری و پنل مدیریت.

## درباره

`bot_1st` یک ربات Python است که با `pyTelegramBotAPI` و MySQL ساخته شده است. این پروژه دو جریان کاری اصلی دارد:

- مشتری: درخواست ساخت ربات یا پروژه، ارسال جزئیات، ارسال توضیحات صوتی و مشاهده وضعیت پروژه.
- ادمین: بررسی سفارش‌ها، مدیریت کاربران و پروژه‌ها، ارسال فایل یا پیام، و استفاده از دستیار هوش مصنوعی OpenAI.

## قابلیت‌ها

- ثبت نام و مدیریت پروفایل مشتری
- ارسال درخواست پروژه با توکن ربات و جزئیات
- پنل مدیریت برای بررسی سفارش‌ها و پشتیبانی
- مدیریت لیست سیاه برای کاربران مشکوک
- استفاده از OpenAI GPT برای ادمین
- ذخیره داده‌ها در MySQL برای مشتریان، محصولات، فروش‌ها و لیست سیاه
- پشتیبانی از پیام صوتی و ارسال فایل پروژه
- ثبت گزارش‌ها در `project.log`

## نیازمندی‌ها

- Python 3.11 یا بالاتر
- سرور MySQL
- نصب بسته‌های Python
- تنظیم متغیرهای محیطی

بسته‌های مورد نیاز در `requirements.txt` فهرست شده‌اند.

## نصب

1. مخزن را کلون کنید.
2. یک محیط مجازی بسازید و فعال کنید:

```powershell
python -m venv venv
.\\venv\\Scripts\\Activate.ps1
```

3. بسته‌های مورد نیاز را نصب کنید:

```powershell
pip install -r requirements.txt
```

## پیکربندی

متغیرهای محیطی زیر را قبل از اجرای ربات تنظیم کنید:

- `db_host` — آدرس میزبان MySQL
- `db_user` — نام کاربری MySQL
- `password` — رمز عبور MySQL
- `db_name` — نام پایگاه داده
- `tel_token` — توکن ربات تلگرام یا بله
- `ai_Token` — توکن API OpenAI
- `proxy_1` — آدرس پروکسی اختیاری
- `proxy_2` — آدرس پروکسی اختیاری

مثال در PowerShell:

```powershell
$env:db_host = "localhost"
$env:db_user = "root"
$env:password = "secret"
$env:db_name = "bot_database"
$env:tel_token = "YOUR_TELEGRAM_OR_BALE_TOKEN"
$env:ai_Token = "YOUR_OPENAI_API_KEY"
```

> مقادیر پیکربندی در `confing.py` از متغیرهای محیطی خوانده می‌شوند.

## راه‌اندازی پایگاه داده

برای ایجاد اسکیمای پایگاه داده، دستور زیر را اجرا کنید:

```powershell
python DDL.py
```

این اسکریپت جداول زیر را ایجاد می‌کند:

- `CUSTOMER`
- `PRODUCT`
- `SALE`
- `SALE_ROW`
- `BLACK_LIST`

## اجرای ربات

ربات را با این دستور اجرا کنید:

```powershell
python main.py
```

ربات پیام‌های ورودی را دریافت می‌کند و به صورت خودکار جریان‌های کار مشتری و ادمین را مدیریت می‌کند.

## ساختار پروژه

- `main.py` — منطق اصلی ربات و هندلرهای پیام
- `confing.py` — پیکربندی محیطی و شناسه‌های ادمین
- `DDL.py` — اسکریپت ساخت جداول پایگاه داده
- `DML.py` — عملیات درج/به‌روزرسانی داده‌ها
- `DQL.py` — عملیات خواندن داده‌ها
- `Text.py` — متن دکمه‌ها و پیام‌های ربات
- `requirements.txt` — وابستگی‌های Python
- `Data/voice/` — فایل‌های صوتی ذخیره‌شده

## نکات

- آدرس API پیش‌فرض در `main.py` برای بله تنظیم شده است.
- شناسه‌های ادمین در `confing.py` در متغیر `ADMIN` ذخیره می‌شوند.
- برای تغییر متن ربات، `Text.py` را ویرایش کنید.
